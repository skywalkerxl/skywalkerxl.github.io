到了这里，可能会有人问：“在this变量中添加属性及方法和在原型对象中添加属性及方法有什么区别呢？”

其实这在第一篇文章中已经有提到，如果是赋值给this的话，没创建一个新的对象就要复制这些属性和方法，而在原型上创建的这些属性和方法则不会重新创建，这样做显而易见的好处就是减少了不必要的浪费。

属性和方法封装

在js中并没有public、private或者protected等关键字,当时我们同样可以巧妙的实现。

在这里我们需要知道一下这几点：
1.在函数内部声明的变量及方法，在函数外部是无法访问到的，这里我们可以把它作为私有变量及私有方法。
2.在函数内部通过this创建的属性和方法，我们在用类创建对象时，每个对象都会拥有一份，并且外部可以访问的到，这里我们可以把它作为公有变量及公有方法。
3.在函数内部通过this创建的方法，不但可以访问到这些方法，我们甚至还可以访问到创建这些对象的类的私有属性和方法，由于这些方法的权利比较大，我们可以称之为特权方法
4.在创建对象时，我们可以通过特权方法，我们可以初始化实例对象的一些属性，因此，我们可以将创建对象时调用的特权方法看成类的构造器

在这里，有点我们需要注意的是，我们通过new创建新对象时，由于在类外面通过点语法执行添加的属性和方法没有执行到，因为是无法获取到的，所以在类外面通过点语法定义的属性以及方法被称为类的静态共有属性和类的静态共有方法，但是类通过prototype创建的属性或者方法，通过new新创建的对象是可以获取到的。
